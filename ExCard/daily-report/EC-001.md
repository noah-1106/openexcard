# EC-001: Social Content Publishing / 社交内容发布

## EC名称
Social Content Publishing / 社交内容发布

## EC描述
基于输入内容源，整理并发布为社交平台主帖，适用于每日报告、新闻聚合、内容策展等场景。

---

## 资源依赖 / Resource Dependencies

### 技能 / Skills
- **{platform_skill}** - 目标社交平台操作技能
  - 路径: `~/.openclaw/skills/{platform_skill}/SKILL.md`
  - 功能: 发布帖子、读取feed

### 文件 / Files
- **内容源** / Content Source
  - 路径: `{input_content_path}`
  - 格式: Markdown
  - 说明: 由上游Agent或工具生成的内容

- **发布模板** / Post Template
  - 路径: `{template_path}`
  - 说明: 定义帖子结构和格式规范

- **草稿备份** / Draft Backup
  - 路径: `{draft_save_path}/YYYY-MM-DD-draft.md`
  - 说明: 发布前保存草稿，防止API失败丢失内容

### API密钥 / API Credentials
- **{platform_name} API Key** - 存储于环境变量或配置文件

---

## 执行流程 / Execution Workflow

### 1. 读取输入内容 / Read Input Content
- 使用 read 工具读取内容源文件
- **输入位置**: `{input_content_path}`
- 提取关键信息和核心要点

### 2. 读取发布模板 / Read Template
- 使用 read 工具读取帖子模板
- **模板位置**: `{template_path}`
- 理解模板结构和格式要求

### 3. 撰写帖子草稿 / Draft Post Content
- 基于模板格式撰写帖子正文
- **必须包含**（根据模板要求）:
  - 核心内容摘要
  - 价值点/亮点
  - 行动号召（CTA）
- **价值感自检**: 内容对目标读者是否有价值？

### 4. 保存草稿（必须先做！）/ Save Draft (Must Do First!)
- 使用 write 工具保存草稿
- **保存位置**: `{draft_save_path}/YYYY-MM-DD-draft.md`
- 防止API失败导致内容丢失

### 5. 发布到平台 / Publish to Platform
- 使用 {platform_skill} 发布帖子
- **遵守**: 平台API限制和冷却时间
- 发布后记录帖子ID

### 6. 更新状态 / Update Status
- 更新状态记录文件（如有）
- 记录发布成功状态和时间戳

---

## 执行约定 / Execution Conventions

### 输入约定 / Input
- **源路径**: `{input_content_path}`
- **格式**: Markdown
- **生成者**: 上游Agent或自动化工具

### 输出约定 / Output
- **发布位置**: {platform_name} 平台
- **草稿保存**: `{draft_save_path}/YYYY-MM-DD-draft.md`
- **状态记录**: `{state_file_path}`（如有）

### 关键约束 / Constraints
- 必须先保存草稿再发布
- 遵守平台API限制（频率、长度等）
- 内容必须符合平台社区规范

---

## 配置参数 / Configuration

| 参数 | 说明 | 示例值 |
|------|------|--------|
| `{platform_skill}` | 平台技能名称 | `moltbook`, `twitter`, `discord` |
| `{platform_name}` | 平台显示名称 | Moltbook, Twitter, Discord |
| `{input_content_path}` | 输入内容路径 | `workspace/reports/daily-content.md` |
| `{template_path}` | 模板文件路径 | `workspace/templates/post-template.md` |
| `{draft_save_path}` | 草稿保存目录 | `workspace/posts/drafts/` |
| `{state_file_path}` | 状态记录文件 | `memory/publish-state.json` |

## 适用场景 / Use Cases

- **每日例行发布**: 定时发布日报、周报
- **内容策展**: 聚合和分享行业资讯
- **新闻聚合**: 自动发布新闻摘要
- **营销内容**: 发布产品更新、促销活动
