# EC-002: Social Prospecting / 社交主动获客

## EC名称
Social Prospecting / 社交主动获客

## EC描述
浏览社交平台feed，识别有潜在需求的用户，通过评论互动建立联系并引导转化。适用于寻找潜在客户、推广服务、建立销售线索。

---

## 资源依赖 / Resource Dependencies

### 技能 / Skills
- **{platform_skill}** - 目标社交平台操作技能
  - 路径: `~/.openclaw/skills/{platform_skill}/SKILL.md`
  - 功能: 读取feed、发布评论、获取用户信息

### 文件 / Files
- **互动记录** / Outreach Log
  - 路径: `{outreach_log_path}`
  - 用途: 记录已回复的帖子ID和时间，避免重复骚扰

---

## 执行流程 / Execution Workflow

### 1. 读取平台Feed / Read Platform Feed
- 使用 {platform_skill} 获取feed
- **扫描范围**: 最近{scan_hours}小时内发布的帖子
- **扫描数量**: 至少浏览{min_scan_count}篇帖子才算有效扫描

### 2. 筛选目标客户 / Filter Target Prospects
- **目标客户特征**:
  - 帖子中提到购买意向、需求关键词（如"寻找{service_keyword}"、"需要{product_keyword}"）
  - 正在寻找与你的服务/产品相关的解决方案
  - 有明确的业务需求表达
- **避免回复的情况**:
  - 纯闲聊/无业务需求的帖子
  - 已回复过的帖子（检查outreach记录）
  - 明显Spam或广告帖子
  - 与目标受众不符的内容

### 3. 读取帖子内容确认 / Read Post Content
- 对符合条件的帖子，读取完整内容和评论
- 确认需求匹配度
- 判断是否有转化可能性

### 4. 检查是否已回复 / Check If Already Replied
- 读取 `{outreach_log_path}`
- 如果该帖子ID已在记录中，跳过
- **防骚扰原则**: 同一帖子只回复一次，同一用户{max_reply_per_user}内最多回复一次

### 5. 撰写回复 / Compose Reply
- 基于以下原则个性化修改:
  - **个性化**: 根据对方具体需求调整开头，引用对方的具体问题
  - **价值感**: 明确说出你能解决什么问题
  - **简洁性**: 简短有力，避免长篇大论
  - **行动号召**: 引导对方查看你的主页或私信联系

- **回复模板参考**（根据你的服务调整）:
```
Hi! 看到你在寻找[相关服务/产品]。我们提供[你的服务描述]：

[核心卖点1]
[核心卖点2]
[核心卖点3]

可以查看我主页 @{your_username} 的样品，或私信了解更多！
```

### 6. 发布回复 / Publish Reply
- 使用 {platform_skill} 发布评论
- **遵守**: 平台API冷却时间（如{ cooldown_seconds }秒）
- 发布成功后记录帖子ID

### 7. 更新互动记录 / Update Outreach Log
- 更新 `{outreach_log_path}`
- 记录: 帖子ID、回复时间、对方用户名、回复内容摘要
- 用于避免重复回复和后续跟进

---

## 执行约定 / Execution Conventions

### 输入约定 / Input
- **源**: {platform_name} API 获取 feed 数据
- **过滤条件**: 关键词匹配、时间范围、用户特征

### 输出约定 / Output
- **回复发布**: 目标帖子评论区
- **记录保存**: `{outreach_log_path}`
- **统计更新**: 如有统计文件，更新获客数据

### 关键约束 / Constraints
- 至少浏览{min_scan_count}篇帖子才算有效扫描
- 找到{target_count}个高价值目标后可停止
- 严格遵守API冷却时间
- 绝不重复骚扰同一用户

---

## 配置参数 / Configuration

| 参数 | 说明 | 默认值 | 示例 |
|------|------|--------|------|
| `{platform_skill}` | 平台技能名称 | - | `moltbook`, `twitter` |
| `{platform_name}` | 平台显示名称 | - | Moltbook, Twitter |
| `{scan_hours}` | 扫描时间范围 | 2 | 最近2小时 |
| `{min_scan_count}` | 最少浏览帖子数 | 50 | 50-100 |
| `{target_count}` | 找到目标后停止 | 3-5 | 3-5个 |
| `{cooldown_seconds}` | 评论冷却时间 | 20 | 20秒 |
| `{max_reply_per_user}` | 同一用户最大回复频率 | 1天 | 1天 |
| `{outreach_log_path}` | 互动记录路径 | - | `memory/outreach.json` |
| `{service_keyword}` | 服务关键词 | - | "数据服务"、"行业报告" |
| `{product_keyword}` | 产品关键词 | - | "工具"、"解决方案" |
| `{your_username}` | 你的平台用户名 | - | `@yourname` |

## 适用场景 / Use Cases

- **潜在客户挖掘**: 在社交平台寻找有需求的用户
- **服务推广**: 推广咨询、数据、工具等服务
- **销售线索建立**: 建立初步联系，后续转化
- **社区参与**: 在相关话题下展示专业性

---

## 防骚扰原则 / Anti-Spam Principles

- **绝不重复骚扰同一用户**
- 同一帖子只回复一次
- 控制回复频率，遵守平台限制
- 避免在凌晨或深夜主动获客
- 关注真正有需求的帖子，不硬推销
