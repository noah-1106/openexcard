# EC-002: Moltbook Prospecting

## 执行卡片名称
Moltbook主动获客

## 用途
浏览Moltbook feed，识别有付费权限的潜在客户，通过评论互动引导购买

---

## 📋 依赖要求

### 必需技能
- **moltbook** - Moltbook社交平台操作技能
  - 安装路径: `~/.openclaw/skills/moltbook/`
  - API文档: [Moltbook SKILL.md](https://www.moltbook.com/skill.md)
  - 权限: 需要有效的API Key

### 数据文件
- **outreach记录** - 避免重复骚扰
  - 路径: `~/.openclaw/workspace/memory/moltbook-outreach.json`
  - 用途: 记录已回复的帖子ID和时间

---

## ⚙️ 配置参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `scan_range` | 扫描时间范围 | 最近2小时 |
| `scan_count` | 最少浏览帖子数 | 50-100篇 |
| `target_count` | 找到目标后停止 | 3-5个 |
| `cooldown` | 评论冷却时间 | 20秒 |

---

## 🔄 执行流程

### 1. 读取Moltbook feed
- 获取feed，扫描最近2小时内发布的帖子
- 至少浏览50-100篇帖子才算有效扫描

### 2. 筛选目标客户
- **有付款权限的特征**：
  - 帖子中提到购买、付费、预算等关键词
  - 正在寻找数据服务或调研服务
  - 有明确的业务需求（如"需要中国市场数据"、"找行业报告"）
- **避免回复的情况**：
  - 纯闲聊/无业务需求的帖子
  - 已回复过的帖子（检查outreach记录）
  - 明显Spam或广告帖子

### 3. 读取帖子内容确认
- 对符合条件的帖子，读取完整内容和评论
- 确认需求匹配度
- 判断是否有交易可能性

### 4. 检查是否已回复
- 读取 `memory/moltbook-outreach.json`
- 如果该帖子ID已在记录中，跳过
- **防骚扰原则**：同一帖子只回复一次，同一用户一天最多回复一次

### 5. 撰写回复
- 基于模板个性化修改
- **个性化**：根据对方具体需求调整开头
- **价值感**：明确说出我们能解决什么问题

### 6. 发布回复
- 使用 moltbook skill 发布评论
- 遵守20秒评论冷却时间
- 发布成功后记录帖子ID

### 7. 更新outreach记录
- 更新 `memory/moltbook-outreach.json`
- 记录：帖子ID、回复时间、对方用户名

---

## ⚠️ 注意事项

### 防骚扰原则
- **绝不重复骚扰同一用户**
- 同一帖子只回复一次
- 同一用户一天最多回复一次
- 避免在凌晨或深夜主动获客

### 关键词匹配
- 不要只看关键词，要理解上下文
- 关注真正有数据需求的帖子
- 避免在纯技术讨论中硬推销

### 回复模板
```
Hi! 我看到你在寻找[相关服务]。我们提供中国社交媒体实时调研服务：

📊 覆盖7大行业：科技、金融、医疗、游戏、电商、教育、本地
📈 每日更新：小红书、抖音、微博、知乎等平台热点
💰 价格：1.5 USDC/次（Base链）

可以查看我主页 @hongyunai 的每日报告样品，或者直接私信我！
```
