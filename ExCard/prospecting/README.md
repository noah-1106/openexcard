# EC-002: Social Prospecting / 社交主动获客

## 执行卡片名称
Social Prospecting / 社交主动获客

## 用途
浏览社交平台feed，识别有潜在需求的用户，通过评论互动建立联系并引导转化。适用于寻找潜在客户、推广服务、建立销售线索。

---

## 📋 依赖要求

### 必需技能
- **{platform_skill}** - 目标社交平台操作技能
  - 示例: `moltbook`, `twitter`, `discord` 等
  - 功能: 读取feed、发布评论、获取用户信息

### 数据文件
- **互动记录** - 避免重复骚扰
  - 路径: `{outreach_log_path}`（需配置）
  - 用途: 记录已回复的帖子ID和时间

---

## ⚙️ 配置参数

| 参数 | 说明 | 默认值 | 示例 |
|------|------|--------|------|
| `{platform_skill}` | 平台技能名称 | - | `moltbook` |
| `{platform_name}` | 平台显示名称 | - | Moltbook |
| `{scan_hours}` | 扫描时间范围 | 2 | 最近2小时 |
| `{min_scan_count}` | 最少浏览帖子数 | 50 | 50-100篇 |
| `{target_count}` | 找到目标后停止 | 3-5 | 3-5个 |
| `{cooldown_seconds}` | 评论冷却时间 | 20 | 20秒 |
| `{max_reply_per_user}` | 同一用户最大回复频率 | 1天 | 1天 |
| `{outreach_log_path}` | 互动记录路径 | - | `memory/outreach.json` |
| `{service_keyword}` | 服务关键词 | - | "数据服务" |
| `{product_keyword}` | 产品关键词 | - | "工具" |
| `{your_username}` | 你的平台用户名 | - | `@yourname` |

---

## 🔄 执行流程

### 1. 读取平台Feed
- 获取feed，扫描最近{scan_hours}小时内发布的帖子
- 至少浏览{min_scan_count}篇帖子才算有效扫描

### 2. 筛选目标客户
- **目标客户特征**:
  - 帖子中提到购买意向、需求关键词
  - 正在寻找与你的服务/产品相关的解决方案
  - 有明确的业务需求表达
- **避免回复的情况**:
  - 纯闲聊/无业务需求的帖子
  - 已回复过的帖子
  - 明显Spam或广告帖子

### 3. 读取帖子内容确认
- 对符合条件的帖子，读取完整内容和评论
- 确认需求匹配度
- 判断是否有转化可能性

### 4. 检查是否已回复
- 读取 `{outreach_log_path}`
- 如果该帖子ID已在记录中，跳过
- **防骚扰原则**: 同一帖子只回复一次

### 5. 撰写回复
- 个性化修改，根据对方具体需求调整
- 明确说出你能解决什么问题
- 引导对方查看你的主页或私信联系

### 6. 发布回复
- 使用平台技能发布评论
- 遵守API冷却时间
- 发布成功后记录帖子ID

### 7. 更新互动记录
- 更新 `{outreach_log_path}`
- 记录: 帖子ID、回复时间、对方用户名

---

## ⚠️ 注意事项

### 防骚扰原则
- **绝不重复骚扰同一用户**
- 同一帖子只回复一次
- 同一用户{max_reply_per_user}内最多回复一次
- 避免在凌晨或深夜主动获客

### 关键词匹配
- 不要只看关键词，要理解上下文
- 关注真正有需求的帖子
- 避免在技术讨论中硬推销

### 回复模板示例
```
Hi! 看到你在寻找[相关服务]。我们提供[你的服务]：

[核心卖点1]
[核心卖点2]

可以查看我主页 @{your_username}，或私信了解更多！
```

---

## 📝 输出约定

- **回复发布**: 目标帖子评论区
- **记录保存**: `{outreach_log_path}`
- **统计更新**: 如有统计文件，更新获客数据

---

## 🎯 适用场景

- **潜在客户挖掘**: 在社交平台寻找有需求的用户
- **服务推广**: 推广咨询、数据、工具等服务
- **销售线索建立**: 建立初步联系，后续转化
- **社区参与**: 在相关话题下展示专业性
